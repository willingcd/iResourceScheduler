#### 集群实时资源状态（接口）

调度时除了依赖上述**静态**集群清单（能力边界），还需要各集群**当前可用**的卡数、节点数，用于判断「是否有集群满足卡数要求」「多机资源是否充足」。该数据来自 K8s 资源接口。

- **接口**：`GET /ai/api/v1/k8s/resource/cardinfos`（见项目内 `api.py` / `state_cardinfo.CARDINFO_PATH`）。
- **返回结构概要**：`data` 下按厂商（如 `Nvidia`、`Ascend`）→ 卡型（如 `L20`、`H20`）→ 使用模式组织；调度器**仅使用 PASS_THROUGH（直通/独占）** 模式下的节点列表，用于「整卡独占」调度。
- **调度器用到的字段**：
  - `data.<厂商>.<卡型>.PASS_THROUGH.passThroughNodes`：节点数组。
  - 每个节点：`nodeName`、`availableCardNum`（该节点当前可用直通卡数）。
- **与集群的对应关系**：用「厂商 + 卡型」与上文「集群资源清单」对应（如 Nvidia+L20 → nv-120，Nvidia+H20 → nv-h20）；对每个集群汇总其下所有节点的 `availableCardNum` 得到该集群当前 **free_gpus**，对 `nodeName` 去重计数得到 **free_nodes**，供后续「是否满足卡数/节点数」判断使用。

**接口返回示例（节选，仅保留调度用到的部分）**：

```json
{
  "code": 200,
  "msg": null,
  "data": {
    "Nvidia": {
      "L20": {
        "PASS_THROUGH": {
          "passThroughNodes": [
            {
              "nodeName": "work1",
              "slots": ["13"],
              "availableCardNum": 1
            }
          ]
        }
      },
      "H20": {
        "PASS_THROUGH": {
          "passThroughNodes": [
            { "nodeName": "work2", "slots": ["0", "1"], "availableCardNum": 2 }
          ]
        }
      }
    }
  }
}
```

（SHARED 模式为共享卡，当前调度策略按整卡独占设计，不参与决策；若后续支持共享卡调度可再接入。）

#### 模型大小 -> 卡数映射策略

- **核心公式**：所需卡数 = `ceil(模型显存需求 / 单卡可用显存)`
- **模型显存需求估算（BF16精度）**：显存需求(GB) = 参数量(B) × 2 × 1.2（1.2为KV Cache和运行时开销系数）

| 模型参数量 | 显存需求估算(BF16) | L20(48GB)卡数 | H20(96GB)卡数 | 910C(128GB)卡数 | 910B3(64GB)卡数 |
|---|---:|---:|---:|---:|---:|
| <7B | ~17GB | 1卡 | 1卡 | 1卡 | 1卡 |
| 8B~14B | ~34GB | 1卡 | 1卡 | 1卡 | 1卡 |
| 15B~32B | ~77GB | 2卡(TP) | 1卡 | 1卡 | 2卡(TP) |
| 33B~72B | ~173GB | 4卡(TP) | 2卡(TP) | 2卡(TP) | 4卡(TP) |
| 73B~140B | ~336GB | 8卡(TP) | 4卡(TP) | 4卡(TP) | 8卡(TP) |
| 141B~280B | ~672GB | 需多机(TP+PP) | 8卡(TP) | 8卡(TP) | 需多机(TP+PP) |
| >280B | >672GB | 需多机(TP+PP) | 需多机(TP+PP) | 需多机(TP+PP) | 需多机(TP+PP) |

#### 调度决策流程

```mermaid
flowchart TD
A[输入: 模型参数量 + 推理引擎 + 架构要求] --> B[估算显存需求]
B --> C{引擎是否要求特定架构?}

C -->|昇腾引擎/昇腾推理模型| D[候选集群: 910C, 910B3]
C -->|NVIDIA引擎/NVIDIA模型| E[候选集群: L20, H20]
C -->|通用| F[候选集群: 全部]

D --> G[计算每个候选集群所需卡数/节点数]
E --> G
F --> G

G --> G2[获取各候选集群实时可用资源<br/>调用 cardinfo 接口，解析 PASS_THROUGH 节点]
G2 --> H{是否有集群满足卡数/节点要求?}
H -->|否| I[任务无法调度，通知]
H -->|是| J{所需卡数 > 单机卡数?}

J -->|是，多机推理| K[标记为多机分布式推理]
J -->|否，单机可满足| L[选择最优集群]

K --> K1{多机资源是否充足?}
K1 -->|否| I
K1 -->|是| K2[分配多机资源<br/>配置TP+PP并行策略]

L --> M{多个集群满足?}
M -->|是| M1[优先选择空闲资源最多的集群<br/>其次选择卡数刚好满足的集群]
M -->|否| M2[选择唯一满足的集群]

K2 --> N[记录调度决策日志]
M1 --> N
M2 --> N

N --> O[输出: 集群ID + 卡类型 + 卡数 + 机器数 + 并行策略]
```

#### 并行策略选择

| 卡数 | 并行策略 | 说明 |
|---:|---|---|
| 1卡 | 无并行 | 单卡推理 |
| 2~8卡(单机) | TP(Tensor Parallelism) | 张量并行，同一机箱内 |
| >8卡(多机) | TP + PP(Pipeline Parallelism) | 张量并行 + 流水线并行，跨机箱；如2×910c(128GB)可支持~248B模型 |

---

#### 多机分布式推理说明

当模型显存需求超过单机容量时（如超大模型需要>8卡），需要跨机分布式推理：

- **910C集群**：2台×8=16卡，单机128GB×8=1024GB，双机可支持最大~2048GB显存需求
- **910B3集群**：2台×8=16卡，单机64GB×8=512GB，双机可支持最大~1024GB显存需求
- **L20集群**：4台×8=32卡，可支持更大规模分布式推理
- **多机场景需配置节点间通信**（昇腾HCCL/NVIDIA NCCL），可能需要人工介入
